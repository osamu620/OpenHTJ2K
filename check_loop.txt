VERTICAL DWT ANALYSIS
fdwt.cpp lines 182-209: fdwt_irrev_ver_sr_fixed inner loops

for (int32_t n = -4 + offset, i = start - 2; i < stop + 1; i++, n += 2) {
  for (int32_t col = 0; col < u1 - u0; ++col) {
    float sum = buf[n][col];      <- buf is POINTER INDIRECTION (buf[n] is sprec_t*)
    sum += buf[n + 2][col];        <- then [col] index - TWO LEVELS OF INDIRECTION
    buf[n + 1][col] = buf[n + 1][col] + fA * sum;
  }
}

PROBLEM: The buf[][] pattern with two levels of indirection:
- buf is an array of pointers: sprec_t **buf
- buf[n] gets a sprec_t * pointer
- buf[n][col] dereferences the pointer with column index
- This prevents compiler vectorization of the inner col loop
- Each buf[n][col] access requires memory address computation
